🔒 Privacy Policy

Last updated: January 2025

1. Overview
Scope

This Privacy Policy applies to all Forcequit NSW PSN services:

  • nswpsn.forcequit.xyz – Reference site, live data, incident map, and logs
  • radio.forcequit.xyz – Radio streaming service
  • pager.forcequit.xyz – Pager message feed
  • NSW PSN Discord Bot – Alert notifications
  • Backend APIs – Data aggregation and delivery services

We collect only the minimum data necessary to operate, secure, and improve these services. We do not sell, rent, or trade your personal data.

2. Data We Collect
Collection

2.1 Server Logs (All Services)

Standard server logs may record:

  • IP address and approximate geolocation (country/region)
  • Browser user-agent and device type
  • Pages visited and API endpoints accessed
  • Date, time, and duration of access
  • Error logs and security flags

2.2 Session Tracking (nswpsn.forcequit.xyz)

The NSW PSN site uses a heartbeat system to track active sessions:

  • A randomly generated session ID stored in your browser's localStorage
  • Which page you are viewing (map, live data, logs, reference pages)
  • Whether the page is actively fetching data
  • IP address for rate limiting

Session data is not linked to personal identity and expires after 5 minutes of inactivity.

2.3 Browser Storage (nswpsn.forcequit.xyz)

Your browser's localStorage may store:

  • Session ID for heartbeat tracking
  • Map layer preferences (dark mode, map type)
  • List of viewed incident IDs (for "already viewed" indicators)
  • Filter preferences (quick filters, search history)

sessionStorage may temporarily store:

  • Viewed incident tracking (cleared when tab closes)
  • Cached API responses for performance

2.4 Radio & Pager Data

The radio and pager services may collect:

  • Listener connection events (IP address, timestamps)
  • Call metadata (talkgroup IDs, capcodes, timestamps, duration)
  • Audio recordings of publicly receivable, non-encrypted transmissions
  • Pager messages including coordinates, agency codes, and message content

2.5 Map Editor & User Incidents

If you use the Map Editor to submit incidents:

  • Email address (for authentication via Supabase)
  • Incident details you submit (title, location, description, coordinates)
  • Incident updates and status changes

2.6 Discord Bot

The Discord bot stores:

  • Server ID and Channel ID for configured alert channels
  • Alert type subscriptions (RFS, BOM, Traffic, Pager, etc.)
  • Pager capcode filters (if configured)
  • Role IDs for mention notifications
  • Message URLs for incident linking

The bot does not store individual user data, message content, or personal information.

3. Third-Party Services
External

3.1 Cloudflare

All traffic is routed through Cloudflare for security and performance. Cloudflare may process:

  • IP addresses and geolocation data
  • TLS fingerprints and connection metadata
  • Bot detection and security scores

See Cloudflare's Privacy Policy.

3.2 Supabase

User authentication and incident storage uses Supabase:

  • Email addresses for Map Editor authentication
  • User-submitted incident data

See Supabase's Privacy Policy.

3.3 External Data Sources

Incident data is aggregated from government and third-party sources:

  • NSW Rural Fire Service (RFS) – Bush fire incidents and Fire Danger Ratings
  • Bureau of Meteorology (BOM) – Weather warnings and radar
  • Live Traffic NSW / Transport for NSW – Incidents, roadwork, floods, fires, major events
  • Waze – Crowdsourced road hazards, police reports, roadwork
  • Endeavour Energy / Ausgrid – Power outages (planned and unplanned)
  • Beachwatch / BeachSafe NSW – Beach conditions and safety
  • Traffic cameras – NSW traffic camera feeds
  • Aviation cameras – Airport webcam feeds
  • News RSS feeds – Emergency-related news from NSW sources

This data is cached temporarily (up to 7 days) and is not modified.

4. Cookies & Tracking
Cookies

We use:

  • Essential cookies for session management
  • Cloudflare security cookies (bot detection, DDoS protection)
  • localStorage for user preferences and session IDs

We do NOT use:

  • Advertising or marketing cookies
  • Third-party analytics (Google Analytics, Facebook Pixel, etc.)
  • Cross-site behavioural tracking
  • Fingerprinting for identification

You can clear localStorage at any time via your browser's developer tools to reset all stored preferences.

5. How We Use Data
Purpose

Collected data is used only for:

  • Operating and delivering the services
  • Displaying aggregate statistics (e.g., "X viewers", "Y incidents")
  • Optimising data refresh intervals based on active usage
  • Rate limiting to prevent abuse
  • Security monitoring and incident investigation
  • Debugging technical issues
  • Improving service reliability

We do not:

  • Use data for advertising or profiling
  • Sell or share data with data brokers
  • Build personal profiles of users

7. Data Sharing
Sharing

We do not share personal data except:

  • When required by Australian law or legal process
  • When responding to valid law enforcement requests
  • When necessary to investigate abuse or security threats
  • With service providers (Cloudflare, Supabase) as necessary to operate

Aggregate, anonymised statistics (viewer counts, incident totals) may be displayed publicly.

8. Your Rights
Rights

You may request:

  • Access – Information about what data we hold related to your IP or account
  • Deletion – Removal of log entries or account data (where technically feasible)
  • Correction – Updates to user-submitted incident data
  • Export – A copy of your data in a standard format

Requests may be refused if:

  • Data is needed for security or legal purposes
  • The request is excessive, repetitive, or unfounded
  • We cannot verify your identity

To make a request, email: legal@forcequit.xyz

9. Security
Security

We implement reasonable security measures including:

  • Cloudflare WAF and DDoS protection
  • HTTPS encryption for all traffic
  • Rate limiting and abuse detection
  • Server hardening and access controls
  • Regular security monitoring

No system is perfectly secure. We cannot guarantee absolute security but take reasonable precautions.

Updates
